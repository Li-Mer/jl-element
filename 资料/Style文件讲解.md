# 现代前端组件库基础样式架构

这三个 CSS 文件构建了现代前端组件库的基础样式架构，分工明确，遵循从「标准化」到「变量定义」再到「全局统筹」的层级关系。

## 1. reset.css —— 样式重置（「地基」）

### 核心作用

抹平不同浏览器的默认样式差异，确保组件库在所有浏览器中视觉表现一致。
不同浏览器（Chrome、Firefox、Safari、Edge）对 HTML 标签的默认样式（如 h1 字号、ul 缩进、body 边距）存在差异，reset.css 会将这些默认样式统一清零或标准化。

### 常见内容

- 移除 `body` 标签的默认 `margin`；
- 设置 `box-sizing: border-box`（核心规则：让元素宽度包含 `padding` 和 `border`，避免布局崩坏）；
- 去除列表（`ul`/`ol`）的默认圆点标记和缩进；
- 去除链接（`a` 标签）的默认下划线；

### 形象比喻

> 就像盖房子前先把地面铲平，不管原来的地基是沙地还是草地，都先铺上一层统一的水泥板，为后续搭建打下一致的基础。

## 2. vars.css —— CSS 变量/设计令牌（「字典」）

### 核心作用

作为整个组件库的样式配置中心，利用 CSS Variables（自定义属性）定义所有「设计令牌 (Design Tokens)」，实现样式的统一管理和灵活切换。
若直接在组件中写死具体样式值（如主题色 `#409eff`），后续修改主题/风格需逐处调整；通过 vars.css 定义变量，组件中引用变量，仅修改该文件即可批量更新整个库的样式风格。

### 核心内容（参考文件内容）

- 基础色板：定义基础颜色变量（如 `--jl-color-white`）、主题色映射（Sass-like map 格式，包含 primary、success 等色系）；
- 场景样式变量：背景色（`--jl-bg-color`）、文本色（`--jl-text-color-primary`）；
- 边框系统：边框宽度、圆角（如 `--jl-border-radius-base`）；
- 字体系统：字体族（`--vk-font-family`）、字号层级；
- 动画参数：过渡时长（`--vk-transition-duration`）；
- 衍生样式生成：通过 `@each` 和 `@for` 循环（PostCSS 语法）自动生成衍生色（如 `--jl-color-primary-light-3`）；

### 形象比喻

> 这是一本样式「字典」或设计「调色盘」。组件开发时，无需直接指定具体样式值（如“我要用红色”），只需引用字典中的变量（如“我要用 vars.css 里定义的 primary 颜色”），实现样式的统一管控。

### 总结

1. reset.css 核心价值是**统一浏览器默认样式**，是组件库跨浏览器一致性的“地基”；
2. vars.css 核心价值是**封装全局样式变量**，是组件库样式灵活可控的“配置中心”；
3. 整体架构遵循「标准化 → 变量定义 → 全局统筹」的层级逻辑，是现代组件库样式设计的最佳实践。
